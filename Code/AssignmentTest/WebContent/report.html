Database Commands

The database commands were implemented within the application for the following sections:

Login Check

"SELECT PASSWORD FROM EE_USER.MLECE_USERS WHERE USERNAME='" + uname + "'"

This command checks if the username and password entered in the login.jsp form matches the username and password stored in the database.

If these values match, the user is returned to the Home page, and if not, returned to the login page.

Sign Up

"SELECT MAX(USERID) FROM MLECE_USERS"

This command checks for the maximum value contained in the user ID column, this value will be incremented by 1 and used for the user ID of the newly created user.

"INSERT INTO EE_USER.MLECE_USERS ( " + "USERID, SURNAME, FIRSTNAME, EMAIL, USERNAME, PASSWORD, COUNTRY )" + "VALUES (" + id + ", '" + surname + "', '" + firstname + "', '" + email + "', '" + uname + "', '" + pword + "', '" + country +"')"

This command creates a new user with the new user ID value, and the input user information.

Read in Items

SELECT * FROM MLECE_ITEMS

This command reads in the items from the items database table.

Bid on Items

"UPDATE MLECE_SALES SET CURRENTBID=" + bid + " WHERE SALEID=" + saleid

This command updates the Sales table with the users current bid

Buy Items

"DELETE FROM MLECE_ITEMS WHERE ITEMID=" itemid

These commands delete the purchased item from the database table.

The SALES table entry is kept for record of sales.

Add Sale Items

"INSERT INTO MLECE_ITEMS (" + sellerid + ", '" + itemname + "', " + itemid + ", '" + itemdescription + "', '" + category + "' ")

"INSERT INTO MLECE_SALES (" + saleid + ", " + itemid + ", " + startingprice + ", NULL + , + NULL )";

These commands place the new item into the database with the correct userid and starting price

Login Details

Login may be completed using any of the aforementioned username and password combinations, or tested by creating an account using the Sign-Up feature. A successful login is indicated by a return to the home page, with an unsuccessful login indicated by the reloading of the login page
